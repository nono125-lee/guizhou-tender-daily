# 贵州统一标讯雷达 Skill

## 目标

一次运行完成图文广告与园林绿化采集、施工标讯粗筛、招标计划采集、重点项目公告关联、统一页面生成、测试和发布。用户只需要访问“标讯雷达”。

## 执行入口

用户说“查询标讯”“标讯粗筛”“跑标讯”“查询公告”或只点名某一栏时，在项目目录执行同一个全量入口：

```bash
cd /Users/nonolee/Documents/标讯
PYTHONPATH=src python3 -m tender_agent.unified_site update --publish
```

排错或只重建页面时使用：

```bash
PYTHONPATH=src python3 -m tender_agent.unified_site build
```

## 固定顺序

1. 运行图文广告与园林绿化共用采集，按 `config/industries/` 独立关键词分类。
2. 运行施工标讯增量采集和资格资质粗筛，并同步共享目录。
3. 运行招标计划智能窗口采集和详情缓存更新。
4. 识别资金来源原文命中“国债”“专项”“中央”“省级”或超长期的项目，并将 `config/priority_projects.json` 中的用户重点项目与图文、绿化、施工公告统一关联。
5. 生成 `/site/opportunities/` 标讯雷达和运行状态。
6. 运行图文/绿化、施工、招标计划和统一工作台测试。
7. 测试通过且工作树没有非 `site/` 改动时，提交 `site/`、推送 `main`、更新 `gh-pages`。若并发部署导致 `gh-pages` 推送报告 `cannot lock ref`，重新获取远端分支；远端 SHA 与本轮 `site` 子树 SHA 一致时按已发布处理，不一致才报错，禁止盲目破坏性重试。

任一采集来源失败时保留原有公开数据并在运行状态中标红。部署成功不能代替采集成功；最后分别核对数据更新时间、`warnings` 和线上页面。

## 重点项目记录

用户发来重要的“项目名称 + 资金来源”时，不只在对话中记住；立即执行：

```bash
PYTHONPATH=src python3 -m tender_agent.priority_watch add \
  --project-name "<项目名称>" \
  --fund-source "<资金来源>"
```

确认记录已写入 `config/priority_projects.json`。同名项目更新资金来源，不重复新增。后续采集到同项目公告时，在“重点关联”首屏显示“用户重点提示”，并保留公告和计划原文链接。

## 页面结构

- `重点关联`：默认首屏，用户指定的重点项目优先强提示，同时自动关联资金来源原文命中“国债”“专项”“中央”“省级”或超长期的项目；完整复用施工粗筛的检索、地区、时间、信息源、报名日期、截止日期和资质要求筛选，并提供带数量的信息源快捷按钮。时间筛选至少包含最近 1 天、最近 3 天和最近 7 天。
- `图文广告`：按图文广告关键词命中的公告；共享采集贵州省公共资源交易云的工程建设、政府采购、其他交易三个栏目，并保留栏目级信息源名称。信息源下拉框包含施工粗筛全部来源和图文广告已有来源，并提供与施工粗筛一致的带数量快捷按钮，按钮与下拉框同步。
- `园林绿化`：按绿化、园林、公园、养护、景观、苗木等独立关键词命中的公告；与图文广告共享上述三个公共资源交易云栏目，但按园林绿化关键词独立分类。信息源下拉框包含施工粗筛全部来源和园林绿化已有来源，并提供同样的快捷按钮。
- `施工粗筛`：经过施工资质规则筛选的公告，并提供地区、时间、信息源、报名日期、截止日期和资质要求下拉筛选；信息源同时提供带数量的快捷按钮，与下拉筛选同步。资质范围包含“电力工程施工总承包”、“承装（修、试）”和“地质灾害防治单位”资质。
- `招标计划`：展示全部招标计划的最新项目版本，并提供地州市、区县、发布时间、预计招标月和完整资金来源快捷筛选；按钮最前依次为“国债”“专项”“中央”“省级”“超长期”，前四项按资金来源原文包含匹配。
- `运行状态`：图文/绿化、施工和招标计划的更新时间、数量、告警和本次运行结果。

统一页面标题固定为红色楷体“标讯雷达”。同一项目的多个计划版本在详细计划页保留历史入口。重点关联按施工公告时间倒序显示，不提供确认、排除或恢复待处理状态。

## 内部能力边界

- `construction-tender-intelligence` 继续负责施工来源、资格栏目匹配、增量状态、失败重试和人工反馈。
- `tender-plan-intelligence` 继续负责 AP1 招标计划、资金来源、详情缓存、版本合并和关联算法。
- `tender-intelligence` 负责图文广告与园林绿化的来源采集、字段边界和去重。
- 本 Skill 负责统一编排、重点项目清单、统一页面、测试门禁和发布。

## 数据与隐私

- 公开页面读取 `site/data/latest.json`、`site/construction/data/latest.json`、`site/tender-plan/data/latest.json` 和 `site/opportunities/data/*.json`。
- 私有数据库、账号、Cookie、详情缓存和运行锁不得进入 Git。
- 登录、验证码、短信、人脸、付费或协议确认不得绕过。
