# 贵州施工标讯 Skill

## 统一入口

本 Skill 现在是统一施工机会工作流的内部能力。用户要求日常更新、同时查看施工粗筛与超长期、生成统一页面或发布时，使用 `guizhou-construction-opportunity-intelligence`；只有施工来源排错、资格规则测试或明确要求单独运行施工采集时才直接执行本 Skill。

## 调度边界

- 本 Skill 只定义施工标讯采集、筛选、发布规则，不负责创建或维持 Codex 定时任务。
- 不得自行创建“每日标讯本机增量更新”等 Codex 自动任务。
- 更新可由用户手动触发，或由用户另行配置的外部 Agent/自动化平台触发。

## 核心规则

1. 读取 `config/industries/construction.json`。
2. 正向关键词只能在公告明确的“资格要求”“资质要求”“特殊资格要求”及其同义完整标题栏目中匹配。
3. 项目名称、项目概况、招标范围、采购内容、采购人、代理机构及公告其他位置命中资质词都不算。
4. 资格栏目命中任意一个施工资质关键词才收录；关键词包含“电力工程施工总承包”、“承装（修、试）”和“地质灾害防治单位”；“承装（修、试）”用于匹配承装、承修、承试电力设施许可要求。
5. 项目名称含“监理”“审计”“招标代理”时直接排除。
6. 保留原公告链接、资格要求原文和具体命中词，便于人工复核。
7. 页面字段、去重、新增颜色、采购人、代理机构、日期格式和人工反馈沿用图文广告板块要求。
8. 列表接口必须逐页采集到末页，不能只读取第一页或默认前100条。
9. 首次运行查询最近7日建立公告ID状态；后续按各来源上次成功时间增量查询，并向前重叠6小时。
10. 所有见过的公告ID都记录处理结果，包括不匹配公告；只有新ID、失败ID和未关联变更公告允许再次读取详情。
11. 每周回查最近14日列表、每月回查最近45日列表，只对遗漏ID读取详情。
12. 来源异常时不推进该来源游标；详情失败进入重试队列，不能标记为已处理。
13. 超过7日的已有项目不重新覆盖，但周/月回查发现的历史遗漏项目可以补录。
14. 已人工确认或排除的网址进入冻结状态：确认记录保留现值，排除记录持续剔除，后续采集不再覆盖。
15. 变更、澄清或答疑公告按项目编码关联已有标讯，只更新未冻结项目中明确出现的新字段。
16. 页面“查阅文件”是招标文件解析待办标记，不是项目重要程度评级。标记保存在原有浏览器本地存储中，提交时生成带 `TENDER_DOCUMENT_REVIEW_JSON` 的 GitHub Issue。
17. 处理“查阅文件”Issue 时，逐项目下载公告明确提供的招标文件及相关附件，再调用 `tender-document-analyzer` Skill；遇到登录、验证码、付费或协议确认时停止自动操作并列为需要人工处理。
18. `site/construction/data/latest.json` 同时作为 `tender-plan-intelligence` 的施工公告输入。公开项应稳定保留 `project_name`、`title`、`published_at`、`buyer`、`url`、`source_name`、`source_notice_id`、`project_code`、`project_content`、`matched_keywords`、`is_new` 等字段；公告明确载明时还应保留 `fixed_asset_code`、`investment_project_code`、`approval` 或 `approval_refs`，供超长期计划关联使用，未载明时留空而不是猜测。
19. 本 Skill 只提供经过施工资质规则筛选的公告数据，不判断资金来源、不复制招标计划采集逻辑。超长期资金识别、计划匹配、重点展示和匹配证据由 `tender-plan-intelligence` 负责。
20. 超长期页面的“重点”表示“超长期计划已出现施工采购或招标公告”；施工页面的“查阅文件”仍表示招标文件解析待办，两者不得复用同一状态或文案。

## 信息源

从 `config/construction_sources.json` 读取独立来源：

- 贵州省公共资源交易云：工程建设、政府采购、其他交易。
- 黔云招采：贵州高速、贵州能源、黔晟国资、贵州水投、主板块。
- 贵州省招标投标公共服务平台。
- 云农商电子招采平台。
- “黔顺云采”集采平台。
- 中烟电子采购平台。
- 军队采购网贵州公告。

信息源在配置中存在不等于采集成功。每次更新都检查 `warnings`，来源异常时保留旧数据。
增量状态保存在 `site/construction/data/collector-state.json`，不得手工删除；
删除后会触发一次最近7日重新建档并增加详情访问量。

## 执行

```bash
PYTHONPATH=src python3 -m tender_agent.construction_site
```

输出目录为 `site/construction/`，公开链接与图文广告板块分开。
每次命令成功生成项目内结果后，必须再把施工粗筛的公开网页、样式、脚本和 `latest.json`
复制到 `/Users/nonolee/Desktop/共享win/标讯/施工粗筛/`。共享副本不得包含
`collector-state.json`、`ztb-state.json`、`feedback-state.json`、账号、Cookie 或其他私密数据。
共享副本写入失败时，命令必须返回失败并明确报错，不得静默跳过。
